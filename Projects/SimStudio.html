Simulation Studio

Making interactive simulation development possible by non-developers

Unity C# Runtime Architecture Educational Technology

Project Overview

As the technical lead, I designed and developed a comprehensive simulation authoring platform that transformed our content creation pipeline. The system enables content writers and graphic artists to independently build reactive educational simulations for dental and nursing students without any developer involvement.

Before this project, each simulation required a developer for its entire duration. This bottleneck has been eliminated as now we have mostly pivoted to using this studio as the primary method for simulation development, replacing previous standalone development processes.

The platform supports complex branching scenarios with multiple interaction types including multiple choice questions, reordering, and debriefs. Content creators can build simulations ranging from simple linear scenarios to complex decision trees, such as the critical care A-E assessment procedure with 25 layers and over 200 nodes.

Technical Architecture

Key Technical Challenges

  • Runtime-only construction: Built a fully dynamic simulation engine that constructs entire experiences at runtime from data, with no reliance on Unity scene setups or prefabs.
  • Cross-platform deployment: Designed the architecture to seamlessly run the same simulations across Windows authoring tools and mobile deployment platforms.
  • Dynamic asset loading: Implemented GLB model loading system that handles 3D assets, images, audio, and video content dynamically based on simulation requirements.
  • Extensible framework: Created modular architecture allowing rapid addition of new interaction types and simulation components.
